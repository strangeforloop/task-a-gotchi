# Completed task grouping + stale-now midnight bug fix

## Key Decisions

- Completed tasks (from any section) move to a single "Done" section at the bottom of the list
- Overdue section now shows only *incomplete* overdue tasks (fixes misleading health tally)
- Today section header shows `completedNonOverdue / totalNonOverdue` fraction
- Done section only renders when there is at least one completed task
- `now` in both WeeklyPlanContext and HabitContext is fixed with a midnight timer: one-shot `setTimeout` fires at the next midnight boundary, updates state, and the `[now]` dep re-arms it automatically

## File / Component Structure

```
src/components/tasks/TaskList.tsx   — 3-section layout: incomplete overdue / incomplete today / done
src/context/WeeklyPlanContext.tsx   — midnight timer added after useState for `now`
src/context/HabitContext.tsx        — same midnight timer (now already destructures setNow)
```
