# Fix: AddBar input row floats above the keyboard instead of being hidden by it

## Key Decisions

- Listen for keyboard show/hide events in AddBar to track keyboard height
- Use keyboardWillShow/keyboardWillHide on iOS (fires before animation), keyboardDidShow/Did on Android
- Apply bottom: 50 + keyboardOffset inline to input row so it floats just above the keyboard
- Remove hardcoded bottom: 50 from inputRow StyleSheet entry (replaced by inline)
- Self-contained change — only AddBar.tsx modified

## File / Component Structure

```
src/components/shared/AddBar.tsx   — add Keyboard + Platform imports, keyboardOffset state, useEffect listener, inline bottom on input row
```

## Implementation Order

1. Add Keyboard and Platform to react-native import
2. Add keyboardOffset state (default 0)
3. Add useEffect with keyboardWillShow/Hide listeners (Platform-aware)
4. Remove bottom from inputRow StyleSheet, apply as inline { bottom: 50 + keyboardOffset }
5. npx tsc --noEmit
6. npm run lint
