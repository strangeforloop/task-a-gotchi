# Make main "+" AddBar create recurring tasks that carry week to week

## Key Decisions

- Tasks added via the main "+" AddBar write to store.templates (recurring) instead of store.oneoffs
- store.templates already persists across week rollovers; no new persistence logic needed
- Task dot color changes from green (one-off) to purple (template) — correct visual signal
- Task ID format changes to today-tmpl-{title} — compatible with existing toggleTask logic
- Time badge shows "8:00 AM" for new tasks (template default) — acceptable
- Only addTask changes; WeeklyModal one-off vs recurring distinction is unchanged

## File / Component Structure

```
src/context/WeeklyPlanContext.tsx   — change addTask to mutate store.templates instead of store.oneoffs
```
